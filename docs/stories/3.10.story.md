## Installation

### Windows
Download the latest `.exe` from [Releases](https://github.com/your-org/ez-flow/releases).

### Linux
```bash
# AppImage
chmod +x EZFlow.AppImage
./EZFlow.AppImage

# Debian/Ubuntu
sudo dpkg -i ezflow_*.deb
```

## Usage

1. **First Run**: Complete the setup wizard to download a Whisper model
2. **Transcribe**: Hold `Ctrl+Shift+Space` (or `Cmd+Shift+Space` on Mac)
3. **Speak**: Your words appear at your cursor

### Keyboard Shortcuts

| Action | Windows/Linux | macOS |
|--------|--------------|-------|
| Push-to-talk | Ctrl+Shift+Space | Cmd+Shift+Space |

### Prerequisites
- Rust 1.75+

### Getting Started

```bash
# Clone the repo
git clone https://github.com/your-org/ez-flow.git
cd ez-flow

# Install dependencies
bun install

# Run in development
bun tauri dev

# Run tests
cargo test
bun test
```

## Code Style

### Rust
- Follow `rustfmt` defaults
- Use `clippy` for linting
- Prefer `thiserror` for error types

### TypeScript/Svelte
- Use Prettier for formatting
- Follow ESLint rules
- Prefer composition over inheritance

## Pull Request Process

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Run tests (`cargo test && bun test`)
5. Commit (`git commit -m 'Add amazing feature'`)
6. Push (`git push origin feature/amazing-feature`)
7. Open a Pull Request

### Issue Templates

```yaml
# .github/ISSUE_TEMPLATE/bug_report.yml
name: Bug Report
description: Report a bug in EZ Flow
labels: ["bug"]
body:
  - type: markdown
    attributes:
      value: |
        Thanks for reporting a bug! Please fill out the form below.

  - type: input
    id: version
    attributes:
      label: EZ Flow Version
      placeholder: "1.0.0"
    validations:
      required: true

  - type: dropdown
    id: os
    attributes:
      label: Operating System
      options:
        - Windows 10
        - Windows 11
        - macOS 12 Monterey
        - macOS 13 Ventura
        - macOS 14 Sonoma
        - Ubuntu 22.04
        - Ubuntu 24.04
        - Other Linux
    validations:
      required: true

  - type: textarea
    id: description
    attributes:
      label: Bug Description
      description: What happened? What did you expect?
    validations:
      required: true

  - type: textarea
    id: steps
    attributes:
      label: Steps to Reproduce
      placeholder: |
        1. Open EZ Flow
        2. Click on...
        3. See error
    validations:
      required: true
```
